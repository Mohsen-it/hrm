**التاريخ:** 2026-07-22
**الحالة:** Phase 1 Complete

## المتطلبات المسبقة (Prerequisites)

1. تثبيت Laravel 13
2. تثبيت PHP 8.3+
3. تثبيت MySQL 8.x أو SQLite
4. تشغيل `composer install`
5. تشغيل `npm install`
6. نسخ `.env.example` إلى `.env`
7. تشغيل `php artisan key:generate`

## سيناريو التحقق 1 — إنشاء وتقديم طلب إجازة

### الخطوات

```bash
# 1. تشغيل الخادم
php artisan serve

# 2. فتح المتصفح على
# http://localhost:8000

# 3. تسجيل الدخول كموظف
# البريد: employee@example.com
# كلمة المرور: password

# 4. الانتقال إلى صفحة طلبات الإجازة
# /leaves

# 5. الضغط على "طلب إجازة جديدة"

# 6. ملء النموذج:
#    - نوع الإجازة: إجازة سنوية
#    - تاريخ البداية: 2026-08-01
#    - تاريخ النهاية: 2026-08-05
#    - السبب: إجازة سنوية

# 7. الضغط على "حفظ كمسودة"

# 8. التحقق من:
#    - تم إنشاء الطلب بحالة "مسودة"
#    - تم توليد رقم طلب LR{YYYYMMDD}-{XXXX}
#    - حساب أيام العمل بشكل صحيح (3 أيام)

# 9. الضغط على "تقديم الطلب"

# 10. التحقق من:
#     - تغيرت الحالة إلى "مقدم"
#     - تم إنشاء سلسلة الموافقات
#     - وصل إشعار لمدير القسم
```

### معايير القبول

- [ ] تم إنشاء الطلب بحالة "مسودة"
- [ ] تم توليد رقم طلب فريد
- [ ] حُسبت أيام العمل بشكل صحيح (باستثناء الجمعة والسبت)
- [ ] عند التقديم: تغيرت الحالة إلى "مقدم"
- [ ] تم إنشاء خطوات الموافقة
- [ ] وصل إشعار للمدير

## حل المشكلات الشائعة (Troubleshooting)

### المشكلة: الطلب لا يُنشأ
**الحل:**
- تحقق من صحة البيانات المدخلة
- تحقق من وجود نوع الإجازة
- تحقق من صلاحيات المستخدم

### المشكلة: الموافقة لا تتم
**الحل:**
- تحقق من أن المستخدم هو المعتمد الصحيح
- تحقق من حالة الطلب
- تحقق من أن الخطوة الحالية مطابقة

### المشكلة: الرصيد لا يُخصم
**الحل:**
- تحقق من أن الحالة = gm_approved
- تحقق من وجود سجل رصيد للمستخدم
- تحقق من سنة الإجازة
