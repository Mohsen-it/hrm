## سياق تقني

### التقنيات المستخدمة

- **إطار العمل الأمامي:** Vue 3 (Composition API) + Inertia.js
- **التنسيق:** Tailwind CSS 4.3
- **البناء:** Vite
- **الأيقونات:** Font Awesome 6
- **إدارة الحالة:** Inertia shared props + Vue 3 reactivity
- **نظام التصميم المرجعي:** `DESIGN.md` (Superhuman-inspired)

### المكونات الحالية

- `resources/js/Components/layout/Sidebar.vue` — المكون المشترك للقائمة الجانبية (سيتم تحديثه).
- `resources/js/Components/layout/AppLayout.vue` — التخطيط الرئيسي الذي يستضيف السايدبار.
- `resources/js/Components/ui/Badge.vue` — مكون Badge مشترك (للإشعارات/العدادات).

### نقاط مجهولة تم بحثها

1. **تقسيم المكونات:** هل يُفكّك السايدبار إلى مكونات فرعية؟
2. **ربط الصلاحيات:** كيف يُبنى عنصر القائمة بناءً على `view-{module}` ديناميكياً؟
3. **سلوك الجوال:** هل يكون السايدبار drawer منزلق أم لوحة قابلة للطي؟

> تم حل جميع النقاط المجهولة في `research.md`.

## المرحلة 0: البحث

تم تنفيذها في `research.md`.

### القرارات الرئيسية

- **تقسيم المكونات:** نعم، سيتم إنشاء `SidebarItem.vue` و `SidebarGroup.vue` كمكونات فرعية داخل `Components/layout/` لأنها تُستخدم داخل السايدبار فقط ولا تُكرر في مكان آخر.
- **ربط الصلاحيات:** عنصر القائمة يستقبل prop اسمه `permission`؛ يتم عرضه فقط إذا كان المستخدم يملك الصلاحية المذكورة عبر `usePage().props.auth.permissions`.
- **سلوك الجوال:** سيكون السايدبار `drawer` منزلق من اليمين يغطي الشاشة جزئياً مع `backdrop`، لأنه الأنسب للتنقل في الواجهات RTL.

## اعتبارات التنفيذ

- **الأمان:** عرض الوحدات حسب `view-{module}` فقط؛ بدون تغيير في الصلاحيات.
- **الأداء:** تجنب إعادة حساب قائمة الوحدات عند كل render باستخدام `computed`؛ عدم إجراء استعلامات DB إضافية.
- **اللغة:** جميع التسميات عبر `__('module.label')`؛ تحديث ملفات الترجمة `ar.json` و `en.json`.
- **RTL:** السايدبار على اليمين افتراضياً؛ الأيقونات لا تحتاج انعكاساً باستثناء الأيقونات الاتجاهية.

## ترتيب التنفيذ المقترح

1. تحديث إعدادات Tailwind (إضافة ألوان DESIGN.md إن لم تكن موجودة).
2. تحديث `Sidebar.vue` بالهيكل الجديد.
3. إنشاء `SidebarItem.vue` و `SidebarGroup.vue`.
4. تحديث `AppLayout.vue` لدعم طي/فتح السايدبار.
5. تحديث ملفات الترجمة.
6. إضافة/تحديث اختبارات الواجهة (Browser/Component Tests).
7. تشغيل `php artisan pint`.
