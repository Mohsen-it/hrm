## 2. Database Reset + Seed (the critical step)

This single command exercises both the migration and all three seeders.

```bash
php artisan migrate:fresh --seed
```

### 2.1 Expected DB state

```bash
php artisan tinker --execute='echo "companies: " . \Modules\Companies\Models\Company::count() . PHP_EOL;'
# Expected: companies: 1

php artisan tinker --execute='echo "branches: " . \Modules\Branches\Models\Branch::count() . PHP_EOL;'
# Expected: branches: 2

php artisan tinker --execute='echo "subordinations: " . \Modules\Subordinations\Models\Subordination::count() . PHP_EOL;'
# Expected: subordinations: 2
```

### 2.2 Inspect seeded rows

```bash
php artisan tinker --execute='foreach (\Modules\Companies\Models\Company::all() as $c) { echo $c->company_code . " => " . $c->company_name . PHP_EOL; }'
# Expected: AIRPORT-ALEPPO => مطار حلب الدولي

php artisan tinker --execute='foreach (\Modules\Branches\Models\Branch::all() as $b) { echo $b->branch_code . " => " . $b->branch_name . " (company_id=" . $b->company_id . ")" . PHP_EOL; }'
# Expected:
#   CIVIL-AVIATION => الطيران المدني (company_id=1)
#   SYRIAN-AIR => الخطوط الجوية السورية (company_id=1)

php artisan tinker --execute='foreach (\Modules\Subordinations\Models\Subordination::orderBy("sort_order")->get() as $s) { echo $s->code . " => " . $s->name_ar . " (id=" . $s->id . ")" . PHP_EOL; }'
# Expected:
#   ALEPPO-AIRPORT => مطار حلب (id=1)
#   LATTAKIA-AIRPORT => مطار اللاذقية (id=2)
```

**Pass criteria:** All three counts and printed values match the expected output.

### 2.3 Idempotency check

Run `migrate:fresh --seed` **twice** (or just `db:seed --class=...` again on the same DB):

```bash
php artisan migrate:fresh --seed
php artisan migrate:fresh --seed
```

After the second run, the counts MUST still be `1 / 2 / 2`. If any count is `2 / 4 / 4` the seeders are **not idempotent** — bug, must be fixed.

### 2.4 FK on `users.subordination_id`

```bash
php artisan tinker --execute='
$cols = collect(\Schema::getColumnListing("users"));
echo $cols->contains("subordination_id") ? "OK" : "MISSING";
echo PHP_EOL;
'
# Expected: OK
```

   - `status`: 1

4. **Expected:** redirect to the index, success flash "تم إنشاء التبعية بنجاح", and the new row appears.

5. Click the delete icon.

## 4. Users Form Integration Walkthrough

### 4.1 Create an employee with a subordination

1. Navigate to `http://127.0.0.1:8000/users/create`.
2. In the "المعلومات التنظيمية" section, locate the new "التبعية" FormSelect.
3. **Expected:** the dropdown shows 2 options: "مطار حلب" and "مطار اللاذقية", with placeholder "— اختر التبعية —".
4. Fill the rest of the form (name, email, etc. — minimum required fields).
5. Select "مطار حلب" as the subordination.
6. Click "Save".
7. **Expected:** success message "تم إنشاء الموظف بنجاح".

Verify in DB:

```bash
php artisan tinker --execute='
$u = \Modules\Users\Models\User::where("email", "the-email-you-used@example.com")->first();
echo "subordination_id: " . ($u->subordination_id ?? "NULL") . PHP_EOL;
echo "subordination.name_ar: " . ($u->subordination->name_ar ?? "—") . PHP_EOL;
'
# Expected:
#   subordination_id: 1
#   subordination.name_ar: مطار حلب
```

### 4.2 Edit an employee's subordination

1. Navigate to the user's edit page.
2. **Expected:** the FormSelect shows "مطار حلب" as the selected value.
3. Change it to "مطار اللاذقية".
4. Save.
5. **Expected:** success message "تم تحديث الموظف بنجاح".

Verify in DB:

```bash
php artisan tinker --execute='

echo "subordination_id: " . $u->subordination_id . PHP_EOL;

#   subordination_id: 2

```

### 4.3 Detach the subordination

1. Edit the same user.
2. Open the "التبعية" dropdown and select the placeholder "— اختر التبعية —" (empty value).
3. Save.
4. **Expected:** success message. The user's `subordination_id` becomes `NULL`.

Verify in DB:

```bash
php artisan tinker --execute='echo \Modules\Users\Models\User::find($id)->subordination_id ?? "NULL";'
# Expected: NULL
```

### 4.4 NULL is allowed (validation test)

Re-attempt step 4.1 with the "التبعية" dropdown left empty. **Expected:** the form saves without a `subordination_id_exists` error.

### 4.5 Invalid value is rejected (validation test)

Impossible via the UI (the dropdown only shows valid values), but verifiable via a direct HTTP call:

```bash
# As an authenticated user, send a POST with an invalid id
curl -X POST http://127.0.0.1:8000/users \
  -H "Accept: application/json" \
  -H "X-Inertia: true" \
  -H "X-CSRF-TOKEN: <token>" \
  -d "name=Test&email=test@example.com&password=password&status=1&subordination_id=99999"
```

**Expected:** `422 Unprocessable Entity` with `errors.subordination_id = ["التبعية المختارة غير موجودة"]`.

### 4.6 Show page

1. Navigate to `http://127.0.0.1:8000/users/{id}` for the user from 4.1.
2. **Expected:** the "التبعية" row shows the Arabic name of the assigned subordination.

### 4.7 Index page

1. Navigate to `http://127.0.0.1:8000/users`.
2. **Expected:** the DataTable has a "التبعية" column showing the subordination name (or "—" if not assigned).
3. **Expected:** a "التبعية" filter dropdown is present above the table.
4. Select a subordination from the filter.
5. **Expected:** the table reloads with only the employees whose `subordination_id` matches.

## 5. Edge Case Tests

### 5.1 Delete a subordination with employees attached

1. Soft-delete a subordination (e.g., `ALEPPO-AIRPORT`) via `tinker` or the UI.
2. **Expected:** the employees that were attached to it now have `subordination_id = NULL` (not deleted).

```bash
php artisan tinker --execute='
\App\Models\User::where("subordination_id", 1)->update(["subordination_id" => null]);
$s = \Modules\Subordinations\Models\Subordination::find(1);
$s->delete();
$stillAttached = \Modules\Users\Models\User::where("subordination_id", 1)->count();
echo "still attached: " . $stillAttached . PHP_EOL;
'
# Expected: still attached: 0
```

### 5.2 Re-run a seeder individually (idempotency at seeder level)

```bash
php artisan db:seed --class="Modules\\Subordinations\\Database\\Seeders\\SubordinationsDatabaseSeeder"
php artisan db:seed --class="Modules\\Subordinations\\Database\\Seeders\\SubordinationsDatabaseSeeder"
```

**Expected:** no exception, count of `subordinations` stays at 2.

### 5.3 Out-of-order seeder run (Branches before Companies)

```bash
# First, clean the DB but keep migrations
php artisan migrate:fresh
# Run Branches before Companies
php artisan db:seed --class="Modules\\Branches\\Database\\Seeders\\BranchesDatabaseSeeder"
```

**Expected:** a warning message (e.g., "CompaniesDatabaseSeeder must run before BranchesDatabaseSeeder. Skipping."), and 0 branches inserted. **No exception thrown.**

### 5.4 Foreign-key safety

Try to insert a `subordination_id` that doesn't exist (bypass the seeder path):

```bash
php artisan tinker --execute='
try {
    \Modules\Users\Models\User::factory()->create(["subordination_id" => 99999]);
    echo "BUG: insert succeeded";
} catch (\Illuminate\Database\QueryException $e) {
    echo "OK: " . substr($e->getMessage(), 0, 50);
}
'
```

**Expected:** an exception (FK violation).

### 5.5 N+1 check (manual)

Run this and count the SQL queries:

```bash
php artisan tinker --execute='
\DB::enableQueryLog();
$users = \Modules\Users\Models\User::with("subordination")->limit(20)->get();
foreach ($users as $u) { $name = $u->subordination?->name_ar; }
echo "queries: " . count(\DB::getQueryLog()) . PHP_EOL;
'
# Expected: queries: 2 (one for users, one for the eager-loaded subordinations)
```

If the count is `1 + N` (e.g., 21 for 20 users), the eager load is missing — bug, must be fixed.

## 10. Rollback / Clean-up

If you need to undo the feature and return to a pre-feature state:

```bash
# 1. Roll back the last two migrations
php artisan migrate:rollback --step=2

# 2. Remove the seeded data manually
php artisan tinker --execute='
\Modules\Subordinations\Models\Subordination::truncate();
\Modules\Branches\Models\Branch::where("company_id", \Modules\Companies\Models\Company::where("company_code", "AIRPORT-ALEPPO")->value("id"))->delete();
\Modules\Companies\Models\Company::where("company_code", "AIRPORT-ALEPPO")->delete();
'

# 3. Remove the new module
rm -rf Modules/Subordinations

# 4. Revert the changes to existing files (Module/Users, Modules/Companies, Modules/Branches, Sidebar)
git checkout -- Modules/Users/ Modules/Companies/ Modules/Branches/ resources/js/Components/layout/Sidebar.vue database/seeders/DatabaseSeeder.php
```

> This is an emergency-only path. The forward path is the supported one.
