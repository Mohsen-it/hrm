## End-to-End Validation

Run on 3 representative pages: `/companies`, `/companies/create`, `/dashboard` (if exists).

1. **Color consistency:** Open browser DevTools → Elements → Computed → check the `background-color` of the primary CTA. Expected: `rgb(250, 82, 15)` (the Mistral orange).

5. **Sunset Stripe Band:** Visible at the bottom of every page.

- [ ] Clicking "Add New" navigates to the create page.

- [ ] Pagination works.

- [ ] RTL toggle (language switcher) re-flows the entire UI.

### Accessibility Checks

- [ ] All form fields have labels.
- [ ] All buttons have accessible names (visible text or aria-label).
- [ ] Tab order is logical (top to bottom, left to right).
- [ ] Focus ring is visible (orange outline).
- [ ] Color contrast meets WCAG AA:
  - Body text on canvas: ink (#1f1f1f) on canvas (#ffffff) = 16.5:1 ✅
  - Primary button text: on-primary (#ffffff) on primary (#fa520f) = 4.51:1 (passes for large text, fails for small)

  - Caption text: steel (#6a6a6a) on canvas = 5.74:1 ✅

### Performance Checks

- [ ] Time to Interactive < 3s.
- [ ] Total JS bundle < 200KB initial.

## Troubleshooting

### Tokens not applying

**Symptom:** Components still look like the old indigo design.
**Fix:** Ensure `app.css` is imported in `vite.config.js` and `@theme` block contains the new tokens. Clear Vite cache: `rm -rf node_modules/.vite`.

### Aliases not working

**Symptom:** Old pages show broken styles after migration.
**Fix:** Confirm aliases are present in `@theme` block:
```css
--color-primary: var(--color-mistral-primary);
```

### RTL broken on new component

**Symptom:** Icon appears on wrong side.
**Fix:** Use logical properties (`ms-2` instead of `mr-2`, `ps-4` instead of `pl-4`).

### Sunset Stripe Band not visible

**Symptom:** Page bottom doesn't show the gradient.
**Fix:** Confirm `<SunsetStripeBand />` is in `AppLayout.vue` outside the main scroll area, with `position: fixed; bottom: 0`.

### Form validation messages not in Arabic

**Symptom:** Error messages show in English.
**Fix:** Update `lang/ar/validation.php` to include translated messages; component uses `t('validation.required')`.
